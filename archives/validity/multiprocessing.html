    Shmuel Kamensky (Business Capital) asked a question.

    June 3, 2019 at 11:49 PM
    Does DemandTools officially support multiple instances running simultaneously?

    If you open multiple instances of the DemandTools executable at once both seem to work fine.

     

    I've also done this programatically with multi-threading which is extremely useful if you have many long running scenarios (since DemandTools is heavily IO/Network bound).

     

    My question is: Was DemandTools build with thread safety in mind?

    e.g.

    does it assume no other processes are writing to the log file or does it properly acquire a lock?

    does it assume no other processes are reading/writing from/to local databases?

    are there any weird issues I can expect to run into if I implement this fully?

      a day ago

      Hi Shmuel,

       

      You can absolutely run multiple instances of DemandTools at once.

       

      However, you can receive record locking errors if your multiple scenarios are editing the same record.

       

      There is no order of operations, so if you want to dedupe first, then update fields, etc. for example, it runs as if it is the only thing running and does not take into account the other instance(s).

       

      It does not assume no other processes are writing to the log file and doesn't lock, etc.

       

      Thanks,

       

      Jennifer

      Hi Shmuel,

       

      It doesn't assume that no other processes are writing to the logfile, but it will write all information to the logfile from each instance.

       

      The logfile will not be locked and will be accurate!
